# PRODUCT

薪跳 PayDance 的产品治理文档。它用于固定产品定位、能力边界与迭代取舍，避免后续版本在“桌面实时工资看板”这一核心体验上摇摆。

## 一句话定位

薪跳 PayDance 是一款属于打工人的桌面实时薪资仪表盘，把今天已经挣到的钱变成一个安静常驻、实时跳动、随时可扫读的桌面看板。

## 用户与场景

- Windows 11 桌面办公用户，希望把工作时间与收入感知变得更直观。
- 喜欢轻量、无账号、本地保存、不上传数据的小工具用户。
- 需要窗口置顶、系统托盘、迷你悬浮金额窗口和低打扰桌面常驻体验的用户。
- 想先通过网页快速理解产品手感，再决定是否下载 Windows 版的潜在用户。

## 核心价值

- **收入可见**：今日入账金额随工作时间持续增长，让每一秒的收入跳动都看得见。
- **扫一眼就懂**：主窗口展示今日入账、已工作、距离下班、今日预计和工作进度。
- **角落也能用**：迷你悬浮窗口只保留金额，适合长期放在屏幕角落。
- **真实工作制**：支持月薪、日薪、时薪、每周工作日、午休剔除和跨零点夜班。
- **本地优先**：薪资与作息配置只保存在本机，不做登录、云同步、遥测或广告。

## 产品边界

- 不开发快捷键功能。
- 不开发提醒功能。
- 不开发分段时间轴。
- 不把迷你悬浮模式改成复杂面板。
- 不把 Web Preview 宣传成完整桌面版；网页端只负责核心体验预览。
- 不在主界面堆叠超过必要层级的信息面。
- 不把设置中心做成营销页；设置中心只承载配置、作者归属和仓库入口。

## 体验原则

- 主窗口负责完整信息，迷你悬浮窗口负责低存在感金额常驻。
- Web Preview 是线上橱窗，负责降低首次体验门槛；Windows 桌面版仍是完整产品。
- 薪资说明和设置中心属于低频入口，不抢主界面注意力。
- 所有错误提示必须简洁、直接、可执行。
- 新功能优先服务“看见每一秒挣了多少钱”，不追求与核心价值无关的复杂度。

## 平台策略

当前正式验证平台是 Windows 11。项目使用 Vue 3、TypeScript 与 Tauri 2，薪资计算逻辑、状态模型和大部分前端界面具备迁移潜力。Web Preview 作为线上体验入口部署到 GitHub Pages，用于预览核心计算、首次配置、设置中心和迷你悬浮的浏览器内模拟效果。Windows 11 版本稳定后，下一阶段的桌面端战略方向是迁移并验证 macOS 版本。

迁移到 macOS、Linux、移动端或小程序时，托盘、置顶、透明窗口、迷你悬浮、系统材质和窗口权限都需要按平台重新设计与验证。项目的跨平台优势是核心逻辑与 Web UI 可复用，不代表当前 Windows 便携版可以直接变成其他平台安装包。
