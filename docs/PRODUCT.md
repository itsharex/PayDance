# PRODUCT

薪跳 PayDance 的产品治理文档。它用来固定产品定位、能力边界和取舍原则，避免后续版本偏离“桌面实时工资看板”这一核心体验。

## 一句话定位

薪跳 PayDance 是一款桌面实时工资看板，把今日入账变成一个安静常驻、实时跳动、随时可扫读的桌面看板。

## 适合谁

- 当前主要 Windows 11 桌面用户，希望把工作时间与收入感知变得更直观。
- 喜欢轻量、无账号、本地保存、不上传数据的小工具用户。
- 需要窗口置顶、系统托盘、迷你悬浮金额窗口和低打扰常驻体验的用户。
- 想先通过网页理解产品手感，再决定是否下载当前官方桌面版的潜在用户。

## 核心价值

- **收入可见**：今日入账金额随工作时间持续增长，让每一秒的收入跳动都看得见。
- **一眼读懂**：主窗口展示今日入账、已工作、距离下班、今日预计和工作进度。
- **角落可用**：迷你悬浮窗口只保留金额，适合长期放在屏幕角落。
- **贴近真实工作制**：支持月薪、日薪、时薪、每周工作日、午休剔除和跨零点夜班。
- **本地优先**：薪资与作息配置只保存在本机，不做登录、云同步、遥测或广告。

## 产品边界

这些方向暂不进入产品范围：

- 快捷键/热键
- 提醒/通知/弹窗
- 历史时间轴、图表或趋势分析
- 打卡、考勤或工时统计
- 云端同步、账号体系或在线服务
- 把迷你悬浮模式扩展成复杂面板
- 把 Web Preview 宣传成完整桌面版

这不是“永远不听建议”，而是为了保护产品的重量级边界。新的功能必须回答一个问题：它是否让用户更轻松地看见“今天正在挣到的钱”？

## 体验原则

- 主窗口负责完整信息，迷你悬浮窗口负责低存在感金额常驻。
- Web Preview 是线上橱窗，用来降低首次体验门槛；Windows 桌面版仍是完整产品。
- 薪资说明和设置中心属于低频入口，不抢主界面注意力。
- 所有错误提示都应简洁、直接、可执行。
- 新功能优先服务“看见每一秒挣了多少钱”，不追求与核心价值无关的复杂度。

## 平台策略

当前官方发布与验证优先级是 Windows 11。项目使用 Vue 3、TypeScript 与 Tauri 2，Web Preview 作为线上体验入口部署到 GitHub Pages，用于预览核心计算、首次配置、设置中心和迷你悬浮的浏览器内模拟效果。

这并不排斥 macOS、Linux 等平台。社区贡献者可以先提交平台适配方案与验证结果；正式发布前仍需明确构建、验证、更新和维护边界。Windows 桌面版目前负责托盘、置顶、透明窗口、迷你悬浮、系统材质和开机自启动等完整能力。
