# PRODUCT

本文件固定薪跳 PayDance 的产品定位、能力边界和取舍原则，防止后续版本偏离核心体验。

## 定位

桌面实时工资看板——把今日入账变成安静常驻、实时跳动、随时可扫读的桌面数字。

## 适合谁

- Windows 11 桌面用户，想让工作时间与收入感知更直观。
- 偏好轻量、无账号、本地保存、不上传数据的小工具用户。
- 需要窗口置顶、托盘常驻、迷你悬浮金额的低打扰体验。
- 想先通过网页试手感，再决定是否下载桌面版的潜在用户。

## 核心价值

- **收入可见**：今日入账随工作时间持续增长，每一秒的收入跳动都看得见。
- **一眼读懂**：主窗口展示今日入账、已工作、距离下班、今日预计和工作进度。
- **角落可用**：迷你悬浮窗只保留金额，适合长期放在屏幕角落。
- **贴近真实工作制**：月薪、日薪、时薪、每周工作日、午休剔除、跨零点夜班。
- **本地优先**：薪资与作息配置只保存在本机，不做登录、云同步、遥测或广告。

## 产品边界

以下方向不在当前产品范围内：

- 快捷键/热键
- 提醒/通知/弹窗
- 历史时间轴、图表或趋势分析
- 打卡、考勤或工时统计
- 云端同步、账号体系或在线服务
- 把迷你悬浮窗扩展成复杂面板
- 把 Web Preview 宣传成完整桌面版

不是拒绝一切建议，而是保护核心边界。新功能必须回答：它是否让用户更轻松地看见”今天正在挣到的钱”？

## 体验原则

- 主窗口负责完整信息，迷你悬浮窗负责低存在感金额常驻。
- Web Preview 是线上橱窗，降低首次体验门槛；桌面版才是完整产品。
- 薪资说明和设置中心属于低频入口，不抢主界面注意力。
- 错误提示简洁、直接、可执行。
- 新功能优先服务”看见每一秒挣了多少钱”，不追求无关的复杂度。

## 平台策略

官方发布与验证优先 Windows 11。Web Preview 部署在 GitHub Pages，用于预览核心计算、首次配置和迷你悬浮的浏览器模拟效果。

不排斥 macOS、Linux 等平台——社区贡献者可以提交适配方案与验证结果，正式发布前需明确构建、验证、更新和维护边界。
