# 治理说明

> [English version →](GOVERNANCE_EN.md)

PayDance 目前由单人维护。维护者对产品边界、发布节奏、许可证、商标和合并决策负责。

## 决策原则

- 先守住核心体验：本地优先、低打扰的桌面实时工资看板。
- 优先接受范围小、能验证、维护成本清楚的改动。
- 不默认加入账号、遥测、云同步，不上传薪资数据。
- Windows 版本质量优先于跨平台扩张。
- 时间追踪、考勤、薪酬系统、个人财务和强提醒类功能，默认不进入主线。

## 贡献路径

1. 小改动可以直接提交 PR；较大功能、平台适配或方向调整，请先开 Issue 说明场景和边界。
2. 第一次贡献建议从 `good first issue` 或 `help wanted` 开始。
3. 提交前阅读 [贡献指南](../.github/CONTRIBUTING.md)，按影响范围提供验证结果。
4. UI、发布链路、安全、许可和品牌相关改动需要维护者明确确认。

## 维护者权限

维护者可以关闭不符合产品边界、风险过高、验证不足、法律来源不清或维护成本过高的 Issue / PR。

如果某个建议值得讨论但暂时不适合实现，维护者会尽量说明原因和可能的轻量替代方案。

## 治理变更

治理规则本身也可以通过 PR 调整。如果未来加入长期维护者，应在授予仓库或发布权限前先更新本文件，写清楚职责、权限和交接方式。
