# 路线图

> [English version →](ROADMAP_EN.md)

薪跳 PayDance 的开发方向。不是发布时间承诺，而是帮助维护者、贡献者和用户理解当前优先级。

## 最近完成

- Web Preview 自动化 QA：本地服务、多视口截图、DOM 与 console 校验。
- 供应链基线治理：`cargo audit`、`cargo deny`、gitleaks、依赖锁定与元数据测试。
- 品牌与许可文档整理：官方素材边界、中文默认文档和英文镜像。
- 发布资产约束：版本化 Windows 文件名、`latest.json`、更新签名与发布后烟测。

## 当前

- 自动更新发布链路复核：密钥轮换演练、`latest.json` 兼容性、便携版更新路径。
- 窗口位置记忆与多显示器恢复。
- 系统时间校准策略：休眠恢复、时区切换、跨天与夜班边界。
- 配置迁移显式版本链，保证后续设置结构调整可追踪、可回滚。
- 首次启动向导示例感知：配置时就能看到更贴近真实收入跳动的预览。

## 接下来

- Authenticode 代码签名，降低 Windows SmartScreen 误报。
- 迷你悬浮窗口右键菜单：透明度、重置位置、恢复主窗口。
- Windows 桌面端原生能力烟雾测试：托盘、置顶、开机自启动、单实例。
- SBOM 生成与发布归档。
- 社区贡献任务标签，让新贡献者更容易找到合适切入点。

## 以后

- 英文官网入口：`/en/`、`hreflang` 与语言切换。
- Playwright 截图回归，用于稳定 Web Preview 首屏和关键状态。
- 可访问性自动检查，例如 axe-core。
- 多币种支持，但必须保持主界面轻量。

## 长期排除方向

以下方向长期不在产品重点内。相关提案需先开 Issue 说明为什么仍服务”桌面实时工资看板”的核心体验：

- 快捷键/热键
- 提醒/通知/弹窗
- 历史时间轴/图表/趋势
- 打卡/考勤/工时统计
- 云端同步/账号/在线服务

路线图的判断依据见 [PRODUCT.md](PRODUCT.md) 与 [贡献指南](../.github/CONTRIBUTING.md)。
