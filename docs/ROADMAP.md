# 路线图

> [English version →](ROADMAP_EN.md)

本文记录薪跳 PayDance 的开发方向。它不是发布时间承诺，而是帮助维护者、贡献者和用户快速理解当前优先级。

## 最近完成

- Web Preview 自动化 QA 已接入 CI：本地服务、多视口截图、DOM、console 与 axe-core 可访问性检查，失败时上传证据。
- 桌面可靠性基础收口：混合时钟会在长时间休眠或系统时间大幅前跳后重新校准；窗口恢复会把完全跑出屏幕的旧坐标拉回可见区域。
- 本地设置可信度提升：配置迁移升级为显式版本链，未来版本配置安全隔离；设置写盘失败会在界面展示轻量提示。
- 更新与发布链路加固：更新器区分开发配置缺失、正式版签名失败、清单异常和网络失败；Release dry-run 增加本次构建产物烟测，发布后烟测不再把 dry-run 误判为公开 Release 验证。
- Web Preview 与可访问性修复：语言切换同步 HTML `lang`，更新角标改为语义按钮。
- 供应链、品牌与许可基线治理：`cargo audit`、`cargo deny`、gitleaks、依赖锁定、元数据测试、官方素材边界和中英文镜像文档。
- 首次体验与设置恢复：移动端首次向导 Footer 固定可见，Web Preview 每次默认打开完整看板；损坏或未来版本设置会静默按项修复并写回。
- 发布与供应链自动化：Rust 核心更新逻辑已有真实单元测试；Release 会执行 Windows EXE 自动启动冒烟、单实例检查并生成 SPDX SBOM；CodeQL 已覆盖 TypeScript 与 Rust，GitHub Actions 全部固定到 Commit SHA。
- 官网发现性：分享图改为三步设置海报，页面标题与结构化数据调整为 Windows 桌面效率工具语义。
- 中英文官网入口：中文保留 `/PayDance/`，英文新增 `/PayDance/en/`，两页拥有独立 SEO 元数据、双向 `hreflang`、自动日期与 Sitemap。
- 维护证据链：Web Preview 已加入固定时间、固定视口的像素差异门禁；桌面 Rust 托盘动作可按行为测试，EXE 冒烟会记录窗口响应与单实例 JSON 结果。
- 贡献者入口：新增中英文架构修改地图，新手任务要求用户可见结果、修改前证据、验收标准和验证命令。

## 当前

- 完成 v0.9.6 发布链路复核：确认 Latest Release、公开资产、`latest.json` 兼容性、便携版自动更新路径与密钥轮换演练。
- 补齐系统时间校准尾项：大幅回拨、时区切换、跨天与夜班边界。
- 完善多显示器恢复：优先保留仍然有效的副屏位置，并提供重置窗口位置入口。
- 让后台更新错误可见：普通网络失败保持低打扰，清单异常与签名验证失败提供明确提示。
- 首次启动向导示例预览：配置时即可看到今日预计、每分钟收入与午休暂停语义。

## 接下来

- Authenticode 代码签名，降低 Windows SmartScreen 误报。
- 迷你悬浮窗口右键菜单：透明度、重置位置、恢复主窗口。
- 继续扩展只能在真实 Windows 会话验证的能力：系统托盘点击、重启后自启动和真实休眠恢复。
- 更新签名真实验签与 gitleaks 下载校验。
- 发布经过产品边界审核的用户可见新手任务，形成公开反馈闭环。

## 以后

- 多币种支持，但必须保持主界面轻量，不引入汇率、税务或财务分析复杂度。

## 长期排除方向

以下方向长期不在产品重点内。相关提案需要先开 Issue 说明为什么仍服务“桌面实时工资看板”的核心体验：

- 快捷键/热键
- 提醒/通知/弹窗
- 历史时间轴/图表/趋势
- 打卡/考勤/工时统计
- 云端同步/账号/在线服务

路线图的判断依据见 [PRODUCT.md](PRODUCT.md) 与 [贡献指南](../.github/CONTRIBUTING.md)。
