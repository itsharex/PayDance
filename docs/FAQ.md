# 常见问题

> [English version →](FAQ_EN.md)

本文回答使用薪跳 PayDance 前后最常见的问题。想了解产品边界，请看 [PRODUCT.md](PRODUCT.md)；遇到故障或想提建议，请看 [SUPPORT.md](SUPPORT.md)。

## 产品与适用场景

### 薪跳 PayDance 是做什么的？

薪跳 PayDance 是一款桌面实时工资看板。配置薪资与上下班时间后，它会把“今天正在挣到的钱”放在桌面上，并随工作时间实时增长。

### 它适合哪些人使用？

它适合主要使用 Windows 11、希望更直观看见工作时间价值的用户。它也适合喜欢轻量、本地保存、无账号、不上传数据的小工具用户。

### 它是工资核算工具、考勤工具，还是时间追踪工具？

都不是。PayDance 的核心是“看见今天正在挣到的钱”，不是管理考勤、记录工时、核算工资单或分析个人财务。

### 为什么不做提醒、打卡、历史图表或账号同步？

这些方向会把产品带向时间管理、考勤或云服务。PayDance 目前优先保持轻量、本地优先、低打扰；完整边界见 [PRODUCT.md](PRODUCT.md)。

## 下载与使用

### 我应该用 Web Preview 还是 Windows 桌面版？

想先体验核心界面和计算逻辑，可以打开 [Web Preview](https://masterbao66.github.io/PayDance/)。想使用托盘、置顶、迷你悬浮、开机自启动等完整桌面能力，请下载 Windows 桌面版。

### Windows 桌面版下载哪个文件？

从 [最新 Release](https://github.com/MasterBao66/PayDance/releases/latest) 下载 `pay-dance-v0.9.4-windows-x64.exe`。Release 页面同时提供 SHA256 校验文件，可用于验证下载完整性。

### 为什么是便携 EXE，而不是安装包？

当前发布策略优先提供便携 EXE，降低安装成本，也减少安装器维护复杂度。你可以直接运行它；需要删除时，删除 EXE 和本地配置即可。

### 如何重新进入首次启动向导？

关闭应用后删除本地配置文件，再重新启动：

```powershell
Remove-Item "$env:APPDATA\com.masterbao.paydance\salary-settings.json"
```

### Web Preview 的设置会影响桌面版吗？

不会。Web Preview 使用浏览器本机 `localStorage`，Windows 桌面版使用 Tauri Store 保存到本机应用数据目录，两者互不影响。

## 薪资与时间计算

### 月薪、日薪、时薪是怎么换算的？

你可以选择月薪、日薪或时薪模式。PayDance 会根据工作日、上下班时间和午休设置，把它们换算成当天收入进度。

### 午休时间会算进收入吗？

取决于你的设置。如果启用午休剔除，午休时段不会计入有效工作时间；如果你的实际薪资规则不扣午休，可以关闭该选项。

### 支持夜班或跨零点工作吗？

支持。PayDance 会处理跨零点夜班场景，让“今天工作进度”和收入跳动保持在合理边界内。

### 今日入账为什么会持续变化？

PayDance 会根据当前时间、有效工作时段和薪资设置计算今日收入，所以金额会在工作时间内持续增长。

### 显示金额是否等于真实到账工资？

不一定。它是基于你输入的薪资与时间设置得到的实时估算，不包含税费、社保、公积金、奖金、请假、加班或公司内部薪资规则。

## 隐私与本地数据

### 薪资数据会上传吗？

不会。PayDance 不需要账号，不上传数据，也不包含遥测。薪资、工作时间和界面偏好只保存在你的设备上。

### 配置保存在哪里？

Windows 桌面版通过 Tauri Store 将配置保存在本机应用数据目录下的 `salary-settings.json`。该文件可能包含个人薪资信息，不建议公开分享。

### 可以删除本地配置吗？

可以。删除 `salary-settings.json` 后，下一次启动会重新进入首次启动向导。删除前请确认你不再需要原配置。

### 项目包含遥测、广告或账号系统吗？

不包含。PayDance 目前没有账号系统、云同步、遥测或广告。

## 桌面能力

### 迷你悬浮窗口怎么用？

在主窗口中双击金额可以进入迷你悬浮模式。迷你悬浮窗口只显示金额，适合放在屏幕角落；双击它可以恢复主窗口。

### 托盘、置顶、开机自启动有什么用？

托盘让应用关闭主窗口后仍可驻留；置顶让看板保持可见；开机自启动适合长期使用。你可以按自己的工作习惯开启或关闭这些能力。

### 为什么某些桌面能力在 Web Preview 里没有？

Web Preview 运行在浏览器里，无法提供系统托盘、窗口置顶、开机自启动等原生桌面能力。这些能力只在 Windows 桌面版中完整提供。

### 多显示器或高 DPI 下显示异常怎么办？

请先记录应用版本、Windows 版本、显示器数量、DPI 缩放和复现步骤，再按 [SUPPORT.md](SUPPORT.md) 提交反馈。截图或录屏会很有帮助。

## 开源、许可与品牌

是。项目代码采用 [AGPL-3.0-only](../LICENSE) 发布，并带有 AGPL 第 7 条允许的附加条款。完整说明见 [LEGAL.md](../legal/LEGAL.md)。

### 可以商用吗？

可以在遵守 AGPL-3.0-only 和项目附加条款的前提下商用。如果你需要闭源集成、OEM、白标或官方品牌授权，请查看 [LEGAL.md](../legal/LEGAL.md) 中的商业授权说明。

### 可以 fork 或修改后发布吗？

可以，但修改版需要保留必要法律声明，明确标注不是官方版本，并使用能清楚区分的名称、图标、应用标识符和发布渠道。

### 为什么修改版不能直接使用官方名称、图标和宣传素材？

这是为了避免用户把第三方修改版误认为官方版本。商标与品牌素材边界见 [TRADEMARK.md](../legal/TRADEMARK.md) 和 [BRAND-ASSETS.md](../legal/BRAND-ASSETS.md)。

## 贡献与反馈

### 我发现 Bug 应该怎么反馈？

请优先使用仓库中的 Bug 反馈表单，并提供版本、系统环境、复现步骤、期望结果和实际结果。不要公开粘贴薪资数据、配置文件、私钥或其他敏感信息。

### 我想提功能建议，应该先看什么？

请先看 [PRODUCT.md](PRODUCT.md)，确认建议是否服务“桌面实时工资看板”的核心体验。接近提醒、打卡、历史图表、账号同步等方向的建议，需要先说明为什么不会让产品变重。

### 开发者可以从哪里开始贡献？

先阅读 [贡献指南](../.github/CONTRIBUTING.md)，再从 `good first issue` 或 `help wanted` 标签的 Issue 入手。文案、文档、测试、发布流程和 Windows 桌面可靠性都是合适切入点。

### 平台适配建议需要提供哪些信息？

请说明目标系统、构建方式、验证范围、人工冒烟项、更新路径和你愿意维护的范围。正式发布前仍需明确构建、验证、更新和维护边界。
