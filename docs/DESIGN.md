# DESIGN

薪跳 PayDance 的视觉规范。约束主界面、迷你悬浮、设置中心、首次启动向导和 Web Preview 的设计方向，避免 UI 迭代时审美反复摇摆。

## 设计基调

- 安静、清爽、克制，有 Windows 11 桌面工具的质感。
- 主视觉服务金额跳动，不添加抢注意力的装饰。
- 避免灰、脏、廉价的质感；深色模式要保持干净的黑色层次。
- 橙色只作为收入、进度和焦点线索，不做大面积装饰。

## 信息层级

1. 今日入账金额。
2. 已工作、距离下班或距离上班、今日预计。
3. 今日进度条。
4. 薪资说明、设置中心、托盘菜单等低频入口。

主界面最多保留两层视觉结构：金额层与看板层。不要继续叠加独立小卡片、浮动徽章或额外说明层。

## 窗口策略

- 主窗口大圆角无边框，避免原生窗口阴影污染四角。
- 顶部状态区和窗口按钮向内、向下留出呼吸空间。
- 设置中心与薪资说明打开后，背景区域和弹层标题栏仍允许拖拽移动窗口。
- 首次启动向导安静、清晰、较高不透明度，减少后方主界面信息透出。
- 迷你悬浮窗只呈现金额与透明度体验，不新增复杂控件。
- Web Preview 用浏览器内舞台承载主窗口，背景安静克制，不喧宾夺主。

## 主题与色彩

- 浅色模式使用干净白底、浅边框和低噪声阴影。
- 深色模式使用接近黑色的干净层次，不使用脏灰、泛白高光或廉价凹槽。
- 进度条保持平面、清晰；圆点可以有克制光感，但不应产生脏、浮、廉价的发光感。
- 主题切换需要同步 Web 与原生窗口主题，避免四角、边框或背景短暂错色。

## 字体与数字

- 主金额跳动数字字体属于稳定资产，非明确需求不更换。
- 看板、设置中心、薪资说明和向导中的数字、英文、符号统一使用看板数字字体。
- `h`、`m`、`¥` 与数字之间保留可感知间距，避免粘连。
- 金额数字是第一视觉焦点，小数可弱化层级，但不能损害金额整体可信度。

## 动效边界

- 金额变化允许克制脉冲光，范围可感知，持续时间短。
- 不添加第二层环境光、不做大面积呼吸光、不让桌面常驻状态变吵。
- 尊重系统"减少动态效果"偏好。

## 组件边界

- 设置中心保持清晰分组，避免拥挤表单。
- 薪资说明只解释日薪、时薪、分薪、秒薪，不承担设置功能。
- 透明度面板小、轻、与迷你悬浮窗居中对齐；右键只负责触发，不改变视觉锚点。
- GitHub 仓库入口像明确按钮，但不在设置中心底部形成笨重横条。
- Web Preview 隐藏托盘、置顶、开机自启动等桌面专属能力，只保留浏览器中可真实体验的交互。

## 回归要求

- 薪资计算、夜班边界和设置变更优先用纯函数测试覆盖。
- 关键 UI 交互优先用 Vue Test Utils 行为测试覆盖。
- 源码字符串测试只保留少量发布约束、视觉防回退和架构边界断言。
- Web Preview 首屏、浅色/深色主题、迷你悬浮和首次向导需持续补充截图回归。
