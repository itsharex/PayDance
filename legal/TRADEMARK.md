# 商标政策

> [English version →](TRADEMARK_EN.md)

本政策用于保护"薪跳 PayDance"作为官方独立产品的来源识别，避免用户把修改版、换皮版或第三方分发版误认为官方版本。本政策不改变 `LICENSE` 授予的代码使用权。

## 受保护标识

以下名称、标识和来源识别元素由项目作者 Mr.Baoboer 保留：

- **"薪跳"**
- **"PayDance"**
- **"薪跳 PayDance"**
- PayDance Logo、应用图标与托盘图标
- 官方宣传海报、社交预览图、官网视觉识别
- 官方截图中可识别的完整产品视觉组合
- 官方 GitHub 仓库、GitHub Pages 官网、Release 下载渠道、更新端点和签名发布链路
- 应用标识符、包元数据和官方发布资产命名中用于识别官方版本的元素

## 允许的事实性使用

无需事先书面许可，你可以：

- 如实说明你的项目"基于 PayDance"、"forked from PayDance"或"兼容 PayDance"
- 链接到官方仓库、官网或 Release 页面
- 在讨论、评测、教程、新闻、学术或安全报告中提及"薪跳"或"PayDance"
- 在保留许可证、版权和来源声明所必需的范围内复制名称

这些使用必须清楚、准确，不得造成官方背书、合作或认证的误解。

## 需要事先书面许可的使用

未经明确书面许可，不得：

- 以"薪跳"、"PayDance"、"薪跳 PayDance"或容易混淆的相似名称发布修改版或衍生产品
- 将官方 Logo、应用图标、托盘图标、海报或官网视觉作为修改版产品标识
- 使用官方发布渠道、更新端点、签名链路或下载资产命名来分发修改版
- 暗示修改版由 Mr.Baoboer 或 PayDance 项目制作、批准、赞助、认证或背书
- 使用 PayDance 品牌推广无关产品、服务、课程、社群或商业活动

## 修改版与 Fork 的最低区分要求

如果你分发修改版、二进制构建、安装包、便携版或在线运行版本，必须确保普通用户可以清楚分辨它不是官方 PayDance。通常应至少替换或修改：

- 产品名称
- 应用图标和托盘图标
- 应用标识符、包名、窗口标题和包元数据
- 官网、下载页、更新端点和签名公钥
- Release 资产命名、安装包名称和自动更新清单
- README、"关于"视图和法律声明中的官方性表述

可接受示例：

- `MySalary（基于 PayDance 修改）`
- `工资助手（forked from PayDance）`
- `Acme Wage Board, based on PayDance`

不可接受示例：

- `PayDance Plus`
- `薪跳 Pro`
- `PayDance CE`
- 使用官方 PayDance 图标但只在角落写一个小字说明的修改版
